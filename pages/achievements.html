Testing		 We automate all Automatable Functional regression as well as sanity test cases . We do have the ability to run any Suite based on the priorities assigned (P0,P1,P2,P3).
Based on the data available, we have achieved at an average about 80% Automation against the Total test cases from the Manual Team in each project.
Some not Automatable scenarios includes - Network Interuptions/ Audio and Image Checks/ Cross Device test cases, etc.
25 API and FE Validations : We use Rest Assured Library to fetch the API responces and compare the expected data from the API and the Data that is vissible in the Front End.
26 Mixpanel: We compare the events that gets trigerred in Base64 Format when any action is done in the Web or App and validate all the mandatory key values that are required.
We make use of Charles through CURL commands programatically and validate all logs in xml format.
27 DFP : We also read the logs through Charles Proxy for the Double Click for Publishers calls that gets trigerred when any Advertisements are played in Players and validate from the Front End.
28 Framework: We use a Hybrid Framework, which is a combination of Java Language , Page Object Model structure and Maven build management tool and using TestNG to run and generate reports in Extent Reports.
We try to use as much as open source technologies like Selenium for Web App Automation and Appium for Mobile App Automation.

We automate all regular browsers in Mobile as well as Web, like Chrome, Firefox in Windows Platform, and Chrome and Safari on Mac OS.
